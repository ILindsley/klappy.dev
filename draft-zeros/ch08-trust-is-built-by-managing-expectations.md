## Projected argument

Every trust failure is an expectations failure. Every single one.

The marriage that broke — expectations weren't managed. The project that failed — expectations weren't aligned. The employee who quit — their expectations diverged from reality and nobody surfaced it. The AI agent that hallucinated — it generated what it expected the answer to be rather than what it observed.

This is the kernel insight: trust isn't built by being trustworthy. Trust is built by managing expectations so precisely that what happens matches what was promised. The gap between expectation and reality is where trust dies.

All four axioms serve this kernel:

- Reality Is Sovereign — because trust collapses when claims don't match reality
- A Claim Is a Debt — because unmet claims are unmet expectations
- Integrity Is Non-Negotiable Efficiency — because shortcuts create expectation gaps
- You Cannot Verify What You Did Not Observe — because unverified claims set false expectations

The essay should make this feel universal, not technical. Open with the most human version: a promise made to someone you love that you didn't keep. Not from malice — from inattention. From assuming it would work out. From not checking whether your expectation of the situation matched theirs.

Then show the same dynamic in teams, in organizations, in AI collaboration. The pattern is always the same. The domain doesn't matter. Trust is built by managing expectations. Full stop.
