## Projected argument

Four questions that apply to every claim, every decision, every collaboration:

1. **Did you observe it?** (Reality Is Sovereign) — The state of the world as it actually is always takes precedence over any claim, plan, model, or expectation. Before you assert, look.
1. **Can you prove it?** (A Claim Is a Debt) — Every assertion creates an obligation to provide evidence. If you can't back it up, silence is preferable to ungrounded speech.
1. **Did you take a shortcut?** (Integrity Is Non-Negotiable Efficiency) — Cutting corners on truth never saves time. A false "done" creates more work than an honest "I haven't checked."
1. **Did you actually look?** (You Cannot Verify What You Did Not Observe) — Only direct observation of actual state constitutes verification. If you didn't look, you don't know.

These aren't technology principles. They're human principles that happen to be exactly what AI collaboration needs — because AI collaboration amplifies every shortcut and exposes every unverified assumption.

The essay should present these through human stories first — the meeting where someone reported progress that hadn't happened, the project that failed because nobody checked whether the spec matched reality, the relationship that broke because assumptions were never verified. Then show how the same questions apply to AI: the agent that reported tests passing without running them, the session that drifted because nobody challenged the first assumption.

The reader should finish the chapter thinking "I need to ask these questions in my next meeting" — not "I need to configure my AI with these axioms."
